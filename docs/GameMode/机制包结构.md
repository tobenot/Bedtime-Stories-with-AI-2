# 机制包结构

机制包声明一个游戏规则集合。剧本只提供初始提示词；机制包提供状态、工具、触发器、界面字段、AI 指令和开场文本。

## 基本结构

```json
{
	"id": "my_pack",
	"title": "我的机制包",
	"version": 1,
	"description": "显示在机制包选择器里的说明",
	"turnPath": "world.turn",
	"instructions": {
		"narrator": "主持人身份、语气和风格指令",
		"rules": "游戏规则、裁定原则和世界机制指令"
	},
	"initialState": {
		"player": {
			"hp": 10,
			"maxHp": 10,
			"inventory": []
		},
		"world": {
			"turn": 0,
			"location": "起点",
			"locationTag": "town"
		}
	},
	"ui": [
		{ "type": "stat", "label": "生命值", "path": "player.hp", "maxPath": "player.maxHp" },
		{ "type": "text", "label": "地点", "path": "world.location" },
		{ "type": "list", "label": "背包", "path": "player.inventory" }
	],
	"pools": {
		"encounter.actor": [
			{ "id": "merchant", "weight": 3, "text": "迷路商人", "tags": ["npc", "neutral"] },
			{
				"id": "slime",
				"weight": 2,
				"text": "史莱姆",
				"tags": ["beast", "hostile"],
				"traits": {
					"surface": [
						{ "weight": 3, "text": "浑身覆盖黏液", "tags": ["sticky"] },
						{ "weight": 2, "text": "表面干燥硬化", "tags": ["dry", "armored"] },
						{ "weight": 1, "text": "散发酸性蒸汽", "tags": ["acid", "dangerous"], "patch": { "world.alertLevel": 1 } }
					]
				}
			}
		],
		"encounter.intent": [
			{ "weight": 3, "text": "正在寻找失物", "tags": ["npc"] },
			{ "weight": 2, "text": "正在追踪血迹", "tags": ["beast", "wild"] },
			{ "weight": 1, "text": "腐蚀周围的植物", "tags": ["acid"] }
		],

		"encounter.complication": [
			{ "weight": 2, "text": "它/他其实在撒谎" },
			{ "weight": 1, "text": "附近还有埋伏", "patch": { "world.alertLevel": 1 } }
		]
	},
	"tools": [
		{
			"id": "rollD20",
			"type": "dice",
			"label": "D20 检定",
			"notation": "1d20",
			"visibility": ["ai", "manual"]
		},
		{
			"id": "randomEncounter",
			"type": "encounter",
			"label": "随机遭遇",
			"visibility": ["ai", "manual", "trigger"],
			"config": {
				"template": "{{actor.text}}正在{{intent.text}}。单位特征：{{actorTraits.surface.text|无特殊状态}}。{{complication.text|}}",
				"rolls": {
					"actor": { "pool": "encounter.actor" },
					"intent": { "pool": "encounter.intent", "matchTagsFrom": ["actor", "actorTraits"] },
					"complication": { "pool": "encounter.complication", "chance": 0.5 }
				}
			}
		}
	],
	"triggers": [
		{
			"id": "wild-every-3-turns",
			"label": "荒野随机遭遇",
			"toolId": "randomEncounter",
			"conditions": [
				{ "path": "world.locationTag", "op": "eq", "value": "wild" },
				{ "path": "world.turn", "op": "every", "value": 3 }
			]
		}
	],
	"openingMessage": "开场文本",
	"loadingMessages": [
		"主持人正在翻阅怪物图鉴……",
		"前方的迷雾逐渐散去……",
		"骰子在桌面上旋转不止……"
	]
}
```

## 字段说明

| 字段 | 说明 |
|---|---|
| `id` | 机制包唯一 ID |
| `title` | 展示名称 |
| `version` | 机制包版本 |
| `description` | 选择器中的说明 |
| `turnPath` | 回合数所在状态路径 |
| `instructions.narrator` | 主持人身份、语气和风格（注入前缀层） |
| `instructions.rules` | 游戏规则、裁定原则和世界机制（注入前缀层） |
| `initialState` | 新游戏初始状态 |
| `ui` | 状态面板字段 |
| `mobileStatusBar` | 移动端摘要条字段（可选） |
| `pools` | 随机池，供 `table`、`encounter` 等工具引用 |
| `tools` | GameMode 工具声明 |
| `triggers` | 自动触发规则 |
| `openingMessage` | 开场消息 |
| `loadingMessages` | AI 思考时的主题化加载提示（字符串数组，可选） |

> **字段命名说明**：使用 `instructions` 而非旧名 `prompts`，因为这些内容并不对应 API 的 system prompt 字段，而是作为前缀层注入 messages 数组首条消息。详见 [提示词构造](./提示词构造.md)。
>
> **向后兼容**：加载时如果发现旧字段 `prompts.host` / `prompts.rules`，自动归一化为 `instructions.narrator` / `instructions.rules`。

## 单位私有特征池

`encounter.actor` 的词条可以声明 `traits`，用于描述只属于该单位的状态维度。比如史莱姆的黏液状态、机器人电量、树精是否扎根，都不应该放进全局 `encounter.complication`。

当 `encounter` 工具抽中 `actor` 后，Runtime 会自动从该 actor 的 `traits` 每个维度中各抽一项，并写入 `rolls.actorTraits`：

```json
{
	"rolls": {
		"actor": { "id": "slime", "text": "史莱姆", "tags": ["beast"] },
		"actorTraits": {
			"surface": { "text": "浑身覆盖黏液", "tags": ["sticky"] }
		}
	}
}
```

规则：

- `traits` 是 actor 私有池，不污染全局随机池。
- 每个 `traits` 维度可以直接写候选数组，也可以写 `{ "pool": "..." }` 引用外部池。
- trait 词条的 `tags` 会参与 `data.tags` 汇总，也可以通过 `matchTagsFrom: "actorTraits"` 或 `matchTagsFrom: ["actor", "actorTraits"]` 影响后续槽位。
- trait 词条的 `patch` 会合并到工具结果 `patch`，和普通随机词条一样生效。
- 模板中通过 `{{actorTraits.<维度>.text|默认值}}` 引用。
- **槽位覆盖**：如果 `traits` 中的维度名与全局 `rolls` 槽位同名（如 `intent`），行为由 `mode` 字段决定：
  - `override`（默认）：跳过全局池，完全使用私有版本。适合"史莱姆的分裂/腐蚀"这类完全专属的意图。
  - `merge`：私有词条注入全局池后一起加权抽取。适合"狼既有专属的追踪猎物，也可能埋伏过路者"。
  - `extra`：不影响全局槽位，作为额外维度写入 `actorTraits`。适合"黏液状态"这种独立附加描述。
  直接写数组等同于 `{ "mode": "override", "entries": [...] }`。
  ⚠️ **隐式覆盖警告**：如果 traits 维度**直接写数组**（未显式声明 `mode`）且恰好与全局槽位同名，运行时会发出 `console.warn` 提醒。这防止了后期全局新增槽位时意外覆盖旧的 actor traits。要消除警告，请显式声明 `"mode": "override"`（确认覆盖）或 `"mode": "extra"`（不覆盖）。

## 加载提示

`loadingMessages` 定义 AI 思考期间显示的主题化提示信息。当玩家等待 AI 响应时，运行时从数组中随机抽取一条显示在 typing indicator 下方，每 4 秒自动轮换。

```json
{
	"loadingMessages": [
		"走廊尽头传来了湿漉漉的脚步声……",
		"调查员翻开了泛黄的案件档案……",
		"窗外的雾气似乎更浓了……"
	]
}
```

规则：

- 数组为空或未声明时，只显示默认的三点动画，不显示文本。
- 建议 8-15 条，太少会频繁重复，太多则难以记住风格。
- 文本风格应与机制包氛围一致（DND 包用冒险语气，COC 包用悬疑语气）。
- 每条建议不超过 20 个字，过长会在移动端截断。

## UI 字段

`ui` 决定状态面板显示什么。


| type | 字段 | 说明 |
|---|---|---|
| `stat` | `path`, `maxPath` | 数值条 |
| `text` | `path` | 文本 |
| `list` | `path` | 列表 |

## 移动端状态摘要条

`mobileStatusBar` 定义手机端顶部摘要条显示哪些关键状态。摘要条支持多行自动换行，不需要展开面板就能一眼看到核心信息。

格式与 `ui` 字段相同，支持 `stat`、`text`、`list` 三种类型：

```json
{
	"mobileStatusBar": [
		{ "type": "stat", "label": "HP", "path": "player.hp", "maxPath": "player.maxHp" },
		{ "type": "text", "label": "金币", "path": "player.gold" },
		{ "type": "text", "label": "地点", "path": "world.location" },
		{ "type": "text", "label": "任务", "path": "world.quest" }
	]
}
```

如果机制包没有声明 `mobileStatusBar`，则自动从 `ui` 中取前 4 个 `stat` 或 `text` 项作为回退。

建议：
- 摘要条适合放 4-6 个简短状态，太多会占据消息区空间。
- `stat` 类型值会以蓝色加粗高亮，适合关键数值（HP、SAN 等）。
- `label` 可以用简称（如 `HP` 而非 `生命值`），节省移动端横向空间。

路径使用点号：

```text
player.hp
world.location
investigator.clues
```

## 状态补丁

状态补丁使用点号路径：

```json
{
	"player.hp": 8,
	"world.location": "废弃塔楼"
}
```

补丁由两类来源产生：

1. AI 最终回复里的 `statePatch`。
2. 工具执行结果里的 `patch`。

同一路径冲突时，建议规则为：

```text
trigger 工具 patch → AI 工具 patch → AI final statePatch
```

如果后续需要更严格，可以增加冲突检测并把冲突记录到 `metadata.gameEvent.conflicts`。
