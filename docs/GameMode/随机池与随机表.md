# 随机池与随机表

随机遭遇不应该只是“抽一段话”。GameMode 的随机工具应支持组合式生成：先抽事件骨架，再抽参与者、地点细节、动机、奖励、危险等级、后续线索和状态影响。

本文设计的是 GameMode 内部随机池系统，服务 `table`、`pool`、`encounter` 等工具类型。

## 核心概念

| 概念 | 说明 |
|---|---|
| `table` | 单张随机表，从 entries 中抽取一项或多项 |
| `pool` | 可复用随机池，供多个工具或表词条引用 |
| `entry` | 随机表中的一个候选项 |
| `subroll` | 抽中某个 entry 后继续抽子表或随机池 |
| `condition` | 词条是否可用的条件 |
| `weight` | 权重，决定被抽中的相对概率 |
| `tags` | 词条标签，用于过滤、组合和 AI 理解 |
| `effects` | 抽中后产生的状态变化、后续触发、标记等 |
| `template` | 用随机结果填充的展示文本 |

## 推荐结构

机制包可以增加 `pools`，工具可以引用这些随机池。

```json
{
	"pools": {
		"encounter.actor": [
			{ "id": "merchant", "weight": 3, "text": "迷路商人", "tags": ["npc", "neutral"] },
			{ "id": "wolf", "weight": 2, "text": "饥饿的狼", "tags": ["beast", "hostile"] },
			{ "id": "cultist", "weight": 1, "text": "低声祈祷的邪教徒", "tags": ["npc", "hostile", "occult"] }
		],
		"encounter.twist": [
			{ "weight": 3, "text": "正在寻找失物" },
			{ "weight": 2, "text": "受了伤，不敢靠近火光" },
			{ "weight": 1, "text": "身上带着通往主线的线索", "patch": { "world.hasLead": true } }
		],
		"encounter.reward": [
			{ "weight": 4, "text": "少量补给" },
			{ "weight": 2, "text": "旧地图的一角", "patch": { "player.inventory": ["旧地图残片"] } },
			{ "weight": 1, "text": "被诅咒的护符", "tags": ["rare", "cursed"] }
		]
	},
	"tools": [
		{
			"id": "randomEncounter",
			"type": "encounter",
			"label": "随机遭遇",
			"visibility": ["ai", "manual", "trigger"],
			"config": {
				"template": "你遇到了{{actor.text}}，它/他{{twist.text}}。可能获得：{{reward.text}}。",
				"rolls": {
					"actor": { "pool": "encounter.actor" },
					"twist": { "pool": "encounter.twist" },
					"reward": { "pool": "encounter.reward", "chance": 0.45 }
				}
			}
		}
	]
}
```

工具结果应包含每个子抽取结果，而不是只包含最终文本。

```json
{
	"toolId": "randomEncounter",
	"type": "encounter",
	"ok": true,
	"displayText": "你遇到了迷路商人，他正在寻找失物。可能获得：旧地图的一角。",
	"data": {
		"rolls": {
			"actor": { "id": "merchant", "text": "迷路商人" },
			"twist": { "text": "正在寻找失物" },
			"reward": { "text": "旧地图的一角" }
		},
		"tags": ["npc", "neutral"],
		"omitted": []
	},
	"patch": {
		"player.inventory": ["旧地图残片"]
	}
}
```

## 随机算法

### 1. 加权随机

最基础算法。每个 entry 有 `weight`。

```text
总权重 = 所有可用词条 weight 之和
随机数 r = [0, 总权重)
按顺序累加 weight
累加值第一次超过 r 的词条即为结果
```

适合绝大多数机制包。优点是简单、可调试、数据量小。GameMode 初期应优先实现这个。

示例：

```json
[
	{ "weight": 6, "text": "普通旅人" },
	{ "weight": 3, "text": "狼群踪迹" },
	{ "weight": 1, "text": "古代遗物" }
]
```

概率分别是：

```text
普通旅人：60%
狼群踪迹：30%
古代遗物：10%
```

### 2. 条件过滤后加权随机

先按 `condition` 过滤，再加权抽取。

```json
{
	"weight": 2,
	"text": "夜间巡逻队",
	"condition": { "path": "world.timeOfDay", "op": "eq", "value": "night" }
}
```

流程：

```text
entries
  ↓
过滤 condition 不满足的词条
  ↓
过滤 tag / input 不匹配的词条
  ↓
对剩余词条执行加权随机
```

### 3. 不放回抽取

用于“抽 2 个不同特征”“生成 3 件不同战利品”。

```json
{
	"pool": "loot.minor",
	"count": 2,
	"replacement": false
}
```

简单实现：

```text
重复 count 次：
  从当前可用池加权抽一个
  抽中后从池中移除
```

数据量较小时够用。

如果以后需要更严格的加权不放回，可以用 `Efraimidis-Spirakis` 算法：

```text
每个候选生成 key = random() ^ (1 / weight)
按 key 从大到小排序
取前 count 个
```

### 4. 洗牌袋 / 抽牌袋

用于减少重复，适合常见遭遇、天气、闲聊事件。

```json
{
	"pool": "road.event",
	"mode": "shuffleBag",
	"bagKey": "roadEvents"
}
```

流程：

```text
如果 bag 为空：
  把所有可用词条按权重展开或加权洗牌后放入 bag
每次从 bag 中取一个
取完前不会重复
```

运行状态保存到：

```js
chat.metadata.gameMode.randomState.bags.roadEvents = {
	remainingIds: ['merchant', 'brokenCart', 'wolfTracks']
}
```

### 5. 保底 / 动态权重

用于稀有事件不要永远不出现。

```json
{
	"id": "rareRelic",
	"weight": 1,
	"pity": {
		"key": "rareRelicMiss",
		"addWeightPerMiss": 1,
		"resetOnHit": true
	}
}
```

流程：

```text
实际权重 = 基础 weight + 未命中次数 * addWeightPerMiss
命中后清零
未命中则次数 +1
```

### 6. 冷却和防重复

用于避免连续出现同类事件。

```json
{
	"id": "wolfPack",
	"weight": 2,
	"text": "狼群逼近",
	"cooldownTurns": 4
}
```

抽取前过滤：

```text
当前回合 - 上次命中回合 < cooldownTurns，则不可用
```

运行状态保存到：

```js
chat.metadata.gameMode.randomState.entries.wolfPack = {
	lastHitTurn: 8,
	hitCount: 2,
	missCount: 5
}
```

### 7. 预算式随机

用于战斗遭遇、战利品、房间生成。

```json
{
	"pool": "monster.forest",
	"budget": {
		"value": 5,
		"costField": "cost"
	},
	"count": { "min": 1, "max": 4 }
}
```

词条：

```json
[
	{ "id": "goblin", "text": "哥布林", "weight": 5, "cost": 1 },
	{ "id": "wolf", "text": "狼", "weight": 3, "cost": 2 },
	{ "id": "ogre", "text": "食人魔", "weight": 1, "cost": 5 }
]
```

流程：

```text
剩余预算 = 5
重复抽取：
  过滤 cost <= 剩余预算 的词条
  加权抽一个
  剩余预算 -= cost
  达到 count 或无可用词条时结束
```

### 8. 分层随机 / 子词条

随机遭遇推荐使用分层结构，而不是一张大表。

```text
遭遇类型
  ├─ NPC
  │   ├─ 身份
  │   ├─ 目标
  │   └─ 态度
  ├─ 怪物
  │   ├─ 种类
  │   ├─ 数量
  │   └─ 当前行为
  └─ 异象
      ├─ 现象
      ├─ 线索
      └─ 代价
```

示例：

```json
{
	"id": "forestEncounter",
	"type": "table",
	"label": "森林遭遇",
	"entries": [
		{
			"id": "npcEncounter",
			"weight": 4,
			"template": "{{npc.name}}正在{{npc.action}}，态度是{{npc.attitude}}。",
			"subrolls": {
				"npc": {
					"name": { "pool": "npc.identity" },
					"action": { "pool": "npc.action" },
					"attitude": { "pool": "npc.attitude" }
				}
			}
		},
		{
			"id": "monsterEncounter",
			"weight": 3,
			"template": "{{count.text}}只{{monster.text}}正在{{behavior.text}}。",
			"subrolls": {
				"monster": { "pool": "monster.forest" },
				"count": { "pool": "monster.count" },
				"behavior": { "pool": "monster.behavior" }
			}
		}
	]
}
```

## 复杂随机遭遇设计

推荐把随机遭遇拆成这些槽位：

| 槽位 | 说明 |
|---|---|
| `category` | 遭遇类型：NPC、怪物、环境、异象、线索、资源 |
| `actor` | 主要对象：商人、野兽、守卫、幽灵 |
| `intent` | 当前目的：求助、追踪、交易、埋伏、逃亡 |
| `complication` | 复杂因素：受伤、撒谎、被诅咒、时间紧迫 |
| `locationDetail` | 场景细节：断桥、雾地、废弃营火 |
| `reward` | 可能收益：补给、线索、金币、关系 |
| `danger` | 可能代价：受伤、暴露、消耗、污染 |
| `followUp` | 后续钩子：任务、敌人、地图标记、倒计时 |

工具定义：

```json
{
	"id": "randomEncounter",
	"type": "encounter",
	"label": "随机遭遇",
	"visibility": ["ai", "manual", "trigger"],
	"config": {
		"template": "{{locationDetail.text}}附近，{{actor.text}}正在{{intent.text}}。但问题是：{{complication.text}}。",
		"rolls": {
			"category": { "pool": "encounter.category" },
			"actor": { "pool": "encounter.actor", "matchTagsFrom": "category" },
			"intent": { "pool": "encounter.intent", "matchTagsFrom": "actor" },
			"complication": { "pool": "encounter.complication" },
			"locationDetail": { "pool": "encounter.location", "tagsFromState": "world.locationTag" },
			"reward": { "pool": "encounter.reward", "chance": 0.4 },
			"danger": { "pool": "encounter.danger", "chance": 0.35 },
			"followUp": { "pool": "encounter.followUp", "chance": 0.25 }
		}
	}
}
```

`matchTagsFrom` 的含义是：上一步抽中的标签会影响当前池过滤或权重。

例如 `category` 抽到 `npc`，`actor` 优先抽 `tags` 包含 `npc` 的词条。

## `chance`

有些子词条不是每次都有。

```json
{
	"pool": "encounter.reward",
	"chance": 0.4
}
```

流程：

```text
先掷一次 0~1 随机数
小于等于 chance 才执行该子抽取
否则记录为 omitted
```

## 标签匹配

标签用于让组合更合理。

```json
{
	"id": "merchant",
	"text": "迷路商人",
	"tags": ["npc", "neutral", "road"]
}
```

请求工具时 AI 可以传入过滤条件：

```json
{
	"toolId": "randomEncounter",
	"input": {
		"tags": ["road"],
		"dangerLevel": "low"
	}
}
```

Runtime 应先合并三类标签：

```text
1. 工具 input.tags
2. 当前 state 派生标签，例如 world.locationTag
3. 已抽中子词条 tags
```

## 状态影响

词条可以声明 `patch` 或 `effects`。

简单状态补丁：

```json
{
	"text": "旧地图的一角",
	"patch": {
		"player.inventory": ["旧地图残片"]
	}
}
```

推荐扩展 `effects`：

```json
{
	"text": "被诅咒的护符",
	"effects": [
		{ "type": "addItem", "path": "player.inventory", "value": "被诅咒的护符" },
		{ "type": "set", "path": "world.curseLevel", "value": 1 },
		{ "type": "addTag", "path": "world.tags", "value": "cursed" }
	]
}
```

初期可以只实现 `patch`，后续再实现 `effects`。

## 模板渲染

`template` 用抽取结果填充展示文本。

```json
{
	"template": "{{actor.text}}正在{{intent.text}}，但{{complication.text}}。"
}
```

如果某个可选槽位未抽中，模板需要支持默认值：

```text
{{reward.text|没有明显奖励}}
```

初期也可以不做复杂模板，由 Runtime 返回结构化结果，交给 `narrator` 写最终叙述。

## 推荐实现优先级

### 第一阶段

1. `pool` 数据结构。
2. `table` 支持 `condition`、`tags`、`subrolls`。
3. `encounter` 工具支持 `rolls` 和 `template`。
4. 统一 `ToolResult.data.rolls`。

### 第二阶段

1. `chance`。
2. `count`。
3. 不放回抽取。
4. `cooldownTurns`。
5. `randomState` 保存命中记录。

### 第三阶段

1. 洗牌袋。
2. 保底权重。
3. 预算式随机。
4. `effects`。
5. 模板默认值和条件段落。

## 最小可行随机遭遇

如果只想先实现一个足够好用的版本，建议支持：

```json
{
	"id": "randomEncounter",
	"type": "encounter",
	"label": "随机遭遇",
	"config": {
		"rolls": {
			"category": { "pool": "encounter.category" },
			"actor": { "pool": "encounter.actor", "matchTagsFrom": "category" },
			"intent": { "pool": "encounter.intent" },
			"complication": { "pool": "encounter.complication", "chance": 0.5 },
			"reward": { "pool": "encounter.reward", "chance": 0.35 }
		}
	}
}
```

这样每次随机遭遇都至少包含：

```text
类型 + 对象 + 意图 + 可选复杂因素 + 可选奖励
```

比单纯抽一段话更可玩，也更适合让 AI 基于结构化结果继续叙述。
