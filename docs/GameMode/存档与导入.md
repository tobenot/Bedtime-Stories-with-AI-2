# 存档与导入

GameMode 需要同时保存玩家可读文本和结构化运行状态。

## 对话文本

对话文本应保证普通导出也可阅读。

示例：

```text
你压低身形靠近营地，但脚下枯枝发出清脆的断裂声。

【检定】
D20 检定：1d20 = 7，难度 12，失败。

【状态变化】
world.alertLevel：0 → 1

【可选行动】
1. 立刻躲进旁边的灌木
2. 装作路过的旅人
3. 抢先制服守卫
```

## 结构化状态

结构化状态保存在：

```js
chat.metadata.gameMode = {
	packId: 'builtin_dnd_adventure',
	state: {},
	triggerState: {}
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| `packId` | 当前机制包 ID |
| `state` | 当前游戏状态 |
| `triggerState` | 触发器运行状态，例如触发次数、上次触发回合 |

## 消息事件元数据

每条 GameMode assistant 消息建议保存：

```js
message.metadata.gameEvent = {
	phase: 'final',
	toolRequests: [],
	toolResults: [],
	triggerResults: [],
	stateChanges: [],
	choices: []
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| `phase` | 最终阶段，通常为 `final` |
| `toolRequests` | AI 请求过的工具 |
| `toolResults` | 工具执行结果 |
| `triggerResults` | 本回合触发器结果 |
| `stateChanges` | 状态变化记录 |
| `choices` | 本轮可选行动 |

## 工具结果保存

工具结果不能只保存为文本。应同时保存结构化数据：

```json
{
	"toolId": "rollD20",
	"type": "dice",
	"label": "D20 检定",
	"reason": "潜行检定",
	"ok": true,
	"displayText": "D20 检定：1d20 = 7",
	"data": {
		"notation": "1d20",
		"rolls": [7],
		"total": 7,
		"difficulty": 12,
		"outcome": "failure"
	},
	"changes": []
}
```

这样后续可以做：

- 回放游戏过程
- 调试 AI 裁定
- 恢复状态
- 统计骰子和随机表结果
- 生成战报或日志

## 导入机制包

游戏模式左侧可以导入 `.json`。

支持：

- 单个机制包对象
- 机制包数组

导入后写入浏览器本地：

```text
bs2_custom_game_packs
```

## 内置包

当前内置：

- `builtin_dnd_adventure`
- `builtin_coc_investigation`

## 兼容旧存档

如果旧存档只有：

```js
chat.metadata.gameMode = {
	packId: 'builtin_dnd_adventure',
	state: {}
}
```

加载时应补齐：

```js
chat.metadata.gameMode.triggerState ||= {}
```

如果旧消息没有 `metadata.gameEvent`，不需要迁移。它们仍然作为普通历史文本参与上下文。
