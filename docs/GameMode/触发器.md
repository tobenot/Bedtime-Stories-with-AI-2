# 触发器

触发器是在玩家发送行动后，由 GameMode 自动检查的规则。触发器属于系统规则，不需要 AI 判断。

## 基本结构

```json
{
	"id": "night-event",
	"label": "夜间事件",
	"toolId": "strangeEvent",
	"conditions": [
		{ "path": "world.locationTag", "op": "eq", "value": "night" },
		{ "path": "world.turn", "op": "every", "value": 2 }
	]
}
```

## 执行时机

```text
玩家发送行动
  ↓
GameMode 增加回合数
  ↓
检查 triggers
  ↓
执行满足条件的 trigger 工具
  ↓
把 triggerResults 放入本回合 AI 上下文
```

触发器在 AI 调用前执行，所以触发器结果可以被本轮 AI 看到。

## 条件操作符

| op | 说明 |
|---|---|
| `eq` | 等于 |
| `neq` | 不等于 |
| `gt` | 大于 |
| `gte` | 大于等于 |
| `lt` | 小于 |
| `lte` | 小于等于 |
| `truthy` | 为真 |
| `falsy` | 为假 |
| `includes` | 包含 |
| `every` | 数值能被 value 整除 |

## 触发器和工具权限

触发器只能执行 `visibility` 包含 `trigger` 的工具。

```json
{
	"id": "randomEncounter",
	"type": "table",
	"label": "随机遭遇",
	"visibility": ["ai", "manual", "trigger"]
}
```

如果触发器引用了不可用工具，应返回失败的 `ToolResult`，并记录到 `metadata.gameEvent.triggerResults`。

## 推荐增强字段

### `once`

只触发一次：

```json
{
	"id": "first-nightmare",
	"toolId": "nightmareTable",
	"once": true,
	"conditions": [
		{ "path": "world.day", "op": "gte", "value": 2 }
	]
}
```

### `cooldownTurns`

触发后冷却若干回合：

```json
{
	"id": "forest-encounter",
	"toolId": "randomEncounter",
	"cooldownTurns": 3,
	"conditions": [
		{ "path": "world.locationTag", "op": "eq", "value": "wild" }
	]
}
```

### `maxRuns`

最多触发次数：

```json
{
	"id": "omen",
	"toolId": "omenTable",
	"maxRuns": 2,
	"conditions": [
		{ "path": "world.sanity", "op": "lte", "value": 5 }
	]
}
```

## 触发器运行状态

建议保存到：

```js
chat.metadata.gameMode.triggerState = {
	'forest-encounter': {
		runCount: 1,
		lastRunTurn: 6
	}
}
```

这样可以支持 `once`、`cooldownTurns` 和 `maxRuns`。

## 与 AI 工具请求的区别

| 类型 | 谁决定 | 何时执行 | 是否回传 AI |
|---|---|---|---|
| `trigger` | 系统规则 | AI 调用前 | 是，本轮直接进入上下文 |
| `toolRequest` | AI | AI 请求后 | 是，通过 Tool Loop 再次调用 AI |
| `manual` | 玩家 | 点击按钮时 | 默认不自动调用 AI，可作为消息记录 |
