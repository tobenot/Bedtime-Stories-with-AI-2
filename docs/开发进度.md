## 三、GameMode — 随机系统

> 详细设计文档：`docs/GameMode/随机池与随机表.md`

### 第一阶段 ✅ 已完成

- [x] `pool` 数据结构
- [x] `table` 支持 `condition`、`tags`、`subrolls`
- [x] `encounter` 工具支持 `rolls` 和 `template`
- [x] `encounter.actor` 支持内联 `traits`（override / merge / extra 三模式）
- [x] 统一 `ToolResult.data.rolls`
- [x] 加权随机（`pickWeightedEntry()`）
- [x] 条件过滤后抽取（`evaluateConditions()` + `filterEntries()`）
- [x] 标签匹配（`matchTagsFrom`）
- [x] `tagsFromState`（从状态路径读取标签做偏好过滤）
- [x] 子词条（`subrolls`，递归支持 max depth=6）
- [x] 模板渲染（`{{path.to.value|fallback}}` 语法，支持数组展开）
- [x] 词条 `patch` 合并

### 第二阶段 ✅ 已完成

- [x] `chance` 概率跳过
- [x] `count`（含 `{min, max}` 随机范围，`normalizeCount()`）
- [x] 不放回抽取（`replacement: false`）

### 第二阶段 — 遗留项

- [ ] 词条级 `cooldownTurns`（防止同一词条连续出现）
- [ ] `randomState` 持久化命中记录（metadata 中字段已预留但无读写逻辑）

### 第三阶段 — 高级随机

- [ ] 洗牌袋（`shuffleBag`）
  - [ ] bag 初始化：按权重展开或加权洗牌后放入 bag
  - [ ] 取完前不重复，用尽后自动重填
  - [ ] `randomState.bags[bagKey].remainingIds` 持久化
- [ ] 保底/动态权重（`pity`）
  - [ ] `addWeightPerMiss`：每次未命中增加的权重
  - [ ] `resetOnHit`：命中后清零
  - [ ] `randomState.entries[id].missCount` 持久化
- [ ] 预算式随机（`budget`）
  - [ ] `budget.value`：总预算
  - [ ] `budget.costField`：词条上的消耗字段
  - [ ] 循环抽取直到预算耗尽或达到 count 上限
- [ ] `effects` 扩展操作
  - [ ] `addItem`：向数组路径追加项目
  - [ ] `set`：设置路径值
  - [ ] `addTag`：向标签数组追加
  - [ ] 区别于 `patch` 的更细粒度状态操作
